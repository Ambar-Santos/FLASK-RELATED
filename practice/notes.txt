Database Operations
- Learning how to work with models in the python shell. 

> Make sure the FLASK_APP environment is set to app.py

        # Linux / macOS
        export FLASK_APP=my_app

        # Windows (CMD)
        set FLASK_APP=my_app

        # Windows (PowerShell)
        $env:FLASK_APP="my_app"

Creating tables in the database 

>>> from app import db
>>> db.create_all()

Creating roles 

from app import Role, User
admin_role = Role(name='Admin')
mod_role = Role(name='Moderator')
user_role = Role(name='User')
user_jhon = User(username='jhon', role=admin_role)
user_susan = User(username='susan', role=user_role)
user_david = User(username='david', role=user_role)

Changes in the database are managed through a database session 
- To prepare objects to be written to the database, they must be added to the session 

 db.session.add(admin_role)
db.session.add(mod_role)
 db.session.add(user_role)
 db.session.add(user_john)
db.session.add(user_jhon)
db.session.add(user_susan)
 db.session.add(user_david)

- Writing objects to the database

>>> db.session.commit()

>>> print(user_jhon.id)
1

= Database sessions are also called transactions
= db.sessions are not related to the session flask object

a data base session can be also rolled back. 
db.session.rollback()
